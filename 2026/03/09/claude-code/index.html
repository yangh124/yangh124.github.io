Claude Code 概述

Claude Code 是一个代理编码工具，可以读取你的代码库、编辑文件、运行命令，并与你的开发工具集成。可在终端、IDE、桌面应用和浏览器中使用。

Claude Code 是一个由 AI 驱动的编码助手，可帮助你构建功能、修复错误和自动化开发任务。它理解你的整个代码库，可以跨多个文件和工具工作以完成任务。

启动一个会话

Claude Code 的三种模式

可以使用 Shift+Tab 切换模式

  1. 规划模式（plan mode on）：只讨论、不修改文件，适合构思
  2. 默认模式（-）：修改文件前一定会询问用户
  3. 自动模式（accept edits on）：自动修改文件
  4. Bypass Permission 模式（bypassPermissions）：跳过所有权限验证，危险！（claude --dangerously-skip-permissions）

使用 ! 前缀的 Bash 模式

Bash 模式特点：

  • 将命令及其输出添加到对话上下文
  • 显示实时进度和输出
  • 支持相同的 Ctrl+B 后台运行以处理长时间运行的命令
  • 不需要 Claude 解释或批准命令
  • 支持基于历史的自动完成：键入部分命令并按 Tab 以从当前项目中的上一个 ! 命令完成
  • 对于快速 shell 操作同时维护对话上下文很有用

Plan Mode

Plan Mode 让 Claude 通过使用只读操作分析代码库来创建计划，非常适合探索代码库、规划复杂更改或安全地审查代码。

Checkpointing

自动跟踪和回退 Claude 的编辑，快速恢复不需要的更改。

Claude Code 自动跟踪 Claude 在工作过程中的文件编辑，允许您快速撤销更改并回退到之前的状态，以防任何事情出现偏差。

自动跟踪

Claude Code 跟踪其文件编辑工具所做的所有更改：

  • 每个用户提示都会创建一个新的 checkpoint
  • Checkpoints 在会话之间持久存在，因此您可以在恢复的对话中访问它们
  • 在 30 天后自动清理（可配置）

回退更改

按两次 Esc（Esc + Esc）或使用 /rewind 命令打开回退菜单。您可以选择恢复：

  • 仅对话：回退到用户消息，同时保留代码更改
  • 仅代码：恢复文件更改，同时保留对话
  • 代码和对话：将两者都恢复到会话中的先前点

注意：使用终端命令创建的文件无法回滚

恢复以前的对话

启动 Claude Code 时，您可以恢复以前的会话：

  • claude --continue 继续当前目录中最近的对话
  • claude --resume 打开对话选择器或按名称恢复
  • claude --from-pr 123 恢复链接到特定拉取请求的会话
  • 从活跃会话内，使用 /resume 切换到不同的对话
  • 会话按项目目录存储。/resume 选择器显示来自同一 git 存储库的会话，包括 worktrees

常见问题

Q: Claude 修改了我的文件，如何撤销？

A: 按两次 Esc 或使用 /rewind 命令选择回退点。

Q: 如何让 Claude 自动批准所有编辑？

A: 使用 claude --dangerously-skip-permissions，但仅限可信项目。

Q: 会话历史在哪里存储？

A: ~/.claude/projects/<project-hash>/ 目录下。

Q: 如何查看 Claude 执行的所有命令？

A: 使用 /tasks 命令查看所有后台任务历史。

Q: Claude 能访问我的整个硬盘吗？

A: 默认只能访问当前工作目录。需要其他目录需通过 MCP 配置。

Q: 如何在后台长时间运行任务？

A: 使用 Ctrl + B 将任务放到后台，/tasks 查看状态。

Q: 如何切换不同的 Claude 模型？

A: 在设置中修改 model 字段，或使用 /model 命令（如果支持）。

安全提示

⚠️ 重要安全注意事项：

  1. 不要 在公共仓库中使用 --dangerously-skip-permissions
  2. 审查 Claude 的所有文件修改，尤其是生产环境
  3. 限制 MCP 服务器的访问权限到最小必要范围
  4. 不要 在对话中分享敏感信息（API 密钥、密码等）
  5. 定期清理 会话历史中的敏感内容
  6. 使用 .gitignore 排除 .claude/ 目录避免提交敏感配置
